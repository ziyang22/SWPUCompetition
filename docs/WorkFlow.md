# 投影法算法工作流程

本文档记录投影法井眼通过能力计算的原始函数工作流程，基于 `python_src/TouYingFa.py` 中的 `Projection2()` 函数。

## 1. 核心函数签名

```python
def Projection2(all_data, Point_3D, Instrument_length=1, Instrument_Radius=0.01,
                begin_deep=None, end_deep=None, num_step=1, if_draw=False)
```

### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| all_data | DataFrame | 井眼轨迹数据（DEPTH, N, E, H 等列） |
| Point_3D | ndarray | 3D 井壁点云，形状 (depth_points, 24, 3) |
| Instrument_length | float | 工具长度（米），默认 1m |
| Instrument_Radius | float | 工具半径（米），默认 0.01m |
| begin_deep | float | 起始深度（米） |
| end_deep | float | 截止深度（米） |
| num_step | float | 步长（米），默认 1m |
| if_draw | bool | 是否绘制可视化图形 |

### 返回值

- 卡点深度或最小半径位置
- 最大可通过半径（最小半径）
- 每个窗口的最大通过半径列表
- 通过方向列表
- 窗口上下平面重心列表
- 总运行时间
- 绘图数据

## 2. 主算法流程

### 2.1 初始化阶段

1. **参数验证**
   - 检查 `begin_deep <= end_deep`
   - 验证 `num_step < Instrument_length`

2. **深度范围确定**
   - 在轨迹数据中查找起始和截止深度对应的索引
   - 计算实际索引步长

3. **数据结构初始化**
   - `P_all`: 存储所有窗口的上平面中心
   - `R_all`: 存储所有窗口的最大内切圆信息
   - `dir_all`: 存储所有窗口的最优投影方向

### 2.2 主循环：滑动窗口

算法使用滑动窗口方式遍历井眼，每次移动 `h_step` 个索引。

```
┌─────────────────────────────────────────────────────┐
│  窗口定义                                           │
│                                                     │
│  i (上端) ──────────► j (下端)                     │
│  ←──────── 工具长度 ───────→                        │
└─────────────────────────────────────────────────────┘
```

#### 窗口中心计算

- **上平面中心** `Pi`：
  - 刚入井时：使用进口轴心
  - 其它情况：使用 `traj[i - step, 1:]`

- **下平面中心** `Pj`：
  - 使用当前深度点的 `traj[j, 1:]`

### 2.3 投影方向生成

1. **主投影方向计算**
   ```python
   d = norm(Pj - Pi)
   n = (Pj - Pi) / d  # 单位向量
   ```

2. **锥形方向集合**
   ```python
   delta = 0.030 / Instrument_length  # 锥角
   angle_step = delta / 8            # 角度步长
   ```
   - 在主投影方向周围生成锥形角度范围内的多个方向
   - 采样数量约 8 × 2 × 12 ≈ 192 个方向

### 2.4 单个方向计算流程

对每个投影方向执行以下步骤：

```
┌────────────────────────────────────────────────────────────┐
│  投影方向 m                                              │
│                                                            │
│  ┌──────────┐    ┌──────────┐    ┌──────────────┐       │
│  │ 线面求交 │ -> │ 3D->2D   │ -> │ 内边界提取   │       │
│  │line_plane│    │point3d   │    │get_closest  │       │
│  └──────────┘    │ to_2d    │    └──────────────┘       │
│                   └──────────┘             │              │
│                                         ▼              │
│                                  ┌──────────────┐       │
│                                  │ 最大内切圆   │       │
│                                  │max_incircle  │       │
│                                  └──────────────┘       │
│                                         │              │
│                                         ▼              │
│                                  ┌──────────────┐       │
│                                  │ 更新最优结果  │       │
│                                  └──────────────┘       │
└────────────────────────────────────────────────────────────┘
```

1. **线面求交** (`line_plane()`)
   - 将上平面的井壁点沿投影方向投影到下平面
   - 直线参数方程与平面方程求交点

2. **3D 到 2D 投影** (`point_3d_to_2d()`)
   - 将 3D 投影点转换到 2D 平面坐标系
   - 以投影点的平均位置为原点

3. **内边界点提取** (`get_closest_points()`)
   - 计算每个点相对于中心的角度（斜率）
   - 按角度分组，在每组中找距离中心最近的点
   - 这些点构成内边界

4. **最大内切圆计算** (`max_incircle()`)
   - 使用网格搜索方法
   - 默认 30×30 网格（900 个候选点）
   - 半径乘 0.98 作为安全系数

5. **最优结果更新**
   - 如果当前圆更大，则更新最优投影方向和最大内切圆

### 2.5 通过性判断

```
┌─────────────────────────────────────────────────┐
│  判断条件                                        │
│                                                 │
│  if 工具半径 > 最大内切圆半径:                   │
│      → 工具无法通过，记录卡点                    │
│  else:                                          │
│      → 工具可以通过，继续下一窗口                 │
└─────────────────────────────────────────────────┘
```

- **卡点输出**：
  - `stuck_point_{depth}m.txt` - 卡点前 5 米数据
  - `final_result_{depth}m.txt` - 最终结果摘要

- **通过输出**：
  - `pass_last_5m_{depth}m.txt` - 最后 5 米数据

## 3. 辅助函数详解

### 3.1 line_plane() - 直线与平面交点

**功能**：计算直线与平面的交点

**数学原理**：
- 直线方程：P = p + t × (a, b, c)
- 平面方程：Ax + By + Cz + D = 0
- 求解参数 t：t = -(A×p[0] + B×p[1] + C×p[2] + D) / (a×A + b×B + c×C)

### 3.2 projection_direction() - 投影方向生成

**功能**：生成主轴方向周围锥形范围内的所有投影方向

**算法**：
1. 如果 delta = 0，只返回主轴方向
2. 计算垂直于主轴的圆锥面上的点
3. 在 z 方向上采样
4. 对每个 z 值，计算圆周上的两个方向向量

### 3.3 point_3d_to_2d() - 3D 到 2D 投影

**功能**：将 3D 点投影到 2D 平面坐标系

**算法步骤**：
1. 计算第一个投影轴：line1 = o - O（归一化）
2. 计算第二个投影轴：line2 = line1 × n（叉积，归一化）
3. 投影：X = Point_O · line1，Y = Point_O · line2

### 3.4 get_closest_points() - 内边界点提取

**功能**：从投影点集中提取内边界点

**算法**：
1. 计算每个点的角度（斜率）
2. 按角度分组
3. 在每组中找距离中心最近的点

### 3.5 max_incircle() - 最大内切圆计算

**功能**：在点集围成的区域内找到最大内切圆

**算法**：
1. 确定边界范围（使用 0.3 系数缩小）
2. 创建 30×30 网格
3. 遍历所有网格点，计算到所有边界点的最小距离
4. 返回最大圆（半径乘 0.98）

## 4. 计算复杂度

### 时间复杂度

- 外层循环：O(n)，n 为深度点数（约 200 个窗口）
- 投影方向数：O(k)，k ≈ 192
- 内边界点提取：O(m)，m 为投影点数（约 96 个点）
- 最大内切圆：O(grid_num²)，30² = 900

**总体复杂度**：O(n × k × (m + grid_num²))

### 空间复杂度

- 轨迹数据：O(n × 27)
- 3D 点云：O(37760 × 24 × 3)
- 投影点缓冲区：O(96 × 3)
